# 작업 착수 전 확인 절차

복잡한 작업(구현·설계·추가·변경·개편 등)을 시작하기 전에 반드시 아래 순서를 지킨다.

> 이 절차는 규칙(프롬프트) 수준으로 준수한다. 여러 파일에 걸친 대규모 작업은
> 네이티브 **Plan Mode**(EnterPlanMode → 계획 승인 → 실행)를 우선 사용한다.
> (과거 task-plan-guard / confirmation-gate 훅의 강제 방식은 오탐 문제로 제거됨)

---

## 필수 순서

### 1단계: 이해 확인
"이렇게 이해했습니다:" 형식으로 요청 내용을 1-3줄로 요약한다.

### 2단계: 작업 목록 리스트업
번호 있는 구체적 단계 목록을 제시한다.

```
1. ...
2. ...
3. ...
```

### 3단계: 사용자 확인 대기
"진행할까요?" 로 확인을 요청하고 **승인 전 코드·파일 수정 금지**.

### 4단계: 승인 후 실행
---

## 예외 — 즉시 실행 가능

| 예외 유형 | 예시 |
|----------|------|
| git 커밋·푸시 | "커밋해줘", "푸시해줘" |
| 내용 요약·설명 | "요약해줘", "설명해줘", "알려줘" |
| 단순 조회 | "읽어줘", "찾아줘", "보여줘" |
| 1줄 단순 수정 | 오탈자 수정, 변수명 변경 1건 |

---

## 왜 필요한가

작업 전 확인 없이 진행하면:
- 의도와 다른 방향으로 구현될 수 있음
- 되돌리기 어려운 변경이 이미 발생한 후 방향 수정 요청이 들어옴
- 사용자가 놓친 엣지케이스를 미리 발견할 기회를 잃음
