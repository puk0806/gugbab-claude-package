# RIPER 워크플로우

> 소스: https://github.com/tom-doerr/riper, https://www.reddit.com/r/ClaudeAI/comments/1kh2mgk/
> 검증일: 2026-06-06

새 기능 구현, 복잡한 리팩터, 아키텍처 변경 등 **30분 이상 소요될 작업**에 적용한다.
단순 버그 수정이나 한 파일 수정은 적용 불필요.

---

## 5단계 개요

```
R → I → P → E → R
조사   아이디어  계획   실행   검토
(읽기만) (제안만) (문서) (코드) (대조)
```

각 단계 전환 시 **사용자 승인** 필요. 이전 단계로 돌아가는 것은 언제든 가능.

---

## 1단계: Research (조사)

**목표**: 현재 상태 파악. 코드 생성·수정 금지.

수행 내용:
- 관련 파일·디렉토리 읽기
- 기존 패턴·컨벤션 파악
- 제약 조건·의존성 확인
- 외부 문서 조사 (필요 시)

산출물: 현재 상태 요약 + 파악된 제약 목록

```
[Research 완료]
현재 상태: ...
파악된 제약: ...
다음 단계(Innovate)로 진행할까요?
```

---

## 2단계: Innovate (아이디어)

**목표**: 접근 방법 제안. 구현 코드 작성 금지.

수행 내용:
- 가능한 접근법 2~3개 제시
- 각 접근법의 장단점·트레이드오프
- 권장 방향 제시 (근거 포함)

산출물: 옵션 비교표 + 권장 방향

```
[Innovate 완료]
옵션 A: ... (장점/단점)
옵션 B: ... (장점/단점)
권장: 옵션 A (이유: ...)
Plan 단계로 진행할까요?
```

---

## 3단계: Plan (계획)

**목표**: 구현 태스크 분해. 아직 코드 작성 금지.

수행 내용:
- 수정/생성할 파일 목록 명시
- 단계별 태스크 번호 목록 (각 태스크 30분 이내)
- 완료 기준(Definition of Done) 명시
- 예상 리스크·엣지 케이스

산출물: 번호 매긴 태스크 목록 + DoD

```
[Plan 완료]
수정 파일: ...
태스크:
  1. ...
  2. ...
완료 기준: ...
Execute 단계로 진행할까요?
```

---

## 4단계: Execute (실행)

**목표**: Plan에서 정의한 태스크를 순서대로 실행.

규칙:
- Plan의 태스크 순서를 따름
- 각 태스크 완료 후 완료 표시
- Plan 범위를 벗어나는 변경이 필요하면 즉시 보고 후 확인 요청
- 태스크 도중 새 문제 발견 시 실행 중단 → 사용자 보고

```
[Execute 진행 중]
✅ 태스크 1 완료
✅ 태스크 2 완료
🔄 태스크 3 진행 중...
```

---

## 5단계: Review (검토)

**목표**: 구현 결과를 Plan과 대조. 이탈 사항 명시.

수행 내용:
- Plan의 태스크 목록 vs 실제 변경 파일 대조
- 완료 기준(DoD) 달성 여부 확인
- Plan에 없던 변경 사항 목록
- 남은 작업·후속 과제 제안

```
[Review 완료]
✅ 달성: 태스크 1, 2, 3
⚠️ 계획 이탈: (없음 / 있으면 명시)
📋 후속 과제: ...
```

---

## 언제 사용하나

| 적합 | 부적합 |
|------|--------|
| 새 기능 구현 (30분 이상) | 단순 오타 수정 |
| 아키텍처 변경 | 한 파일 내 수정 |
| 다수 파일에 걸친 리팩터 | 문서 작성 |
| 기술 스택 추가 | 설정 값 변경 |

---

## /create-plan 커맨드와의 관계

`/create-plan`은 RIPER의 **Plan 단계**를 단독 실행하는 단축키다.
전체 RIPER 흐름이 필요 없을 때 Plan만 빠르게 뽑아낼 수 있다.
