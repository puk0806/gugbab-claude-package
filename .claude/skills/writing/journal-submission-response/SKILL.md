## 0. 빠른 의사결정 트리

```
[투고 단계 진입] 어디에 있는가?

 ├─ 투고 직후 결과를 기다리는 중 → §5 심사 결과 유형 이해
```

## 1. 투고 전 준비 체크리스트

| 항목 | 확인 방법 |
|------|-----------|
| 학술지 scope 일치 | 학술지 "Aims & Scope" 페이지 정독, 최근 3년 게재 논문 5편 이상 훑어보기 |
| KCI 등재 여부 | https://www.kci.go.kr/ 등재학술지 검색 (등재 / 등재후보 / 미등재 구분) |
| 영문 학술지 indexing | DOAJ, SSCI/SCOPUS, JCR 등재 여부 (해당 학술지 publisher 페이지에서 확인) |
| 분량·형식 규정 | "Instructions for Authors" / "투고 규정" 정독 — 분량(어절·단어 수), 인용 형식(APA/Chicago), 키워드 갯수 |
| 표절 검사 | **투고 전** 카피킬러(국내) 또는 Turnitin(국제)으로 사전 검사 |
| Predatory journal 회피 | §2 참고 |
| 저자 자격 | ICMJE 저자 4기준(아래) 충족 — 이름만 올리는 것 금지 |
| 윤리 선언 준비 | 동시 투고 없음, 모든 저자 동의, AI 활용 명시 (§4) |

**ICMJE 저자 4기준 (모두 충족해야 저자):**
1. 연구 설계·자료 수집·분석·해석에 실질적 기여
2. 논문 초고 작성 또는 비판적 수정
3. 최종본 승인
4. 모든 측면에 대한 책임 동의

> 단순 자료 제공자, 자금 지원자, 부서장은 저자가 아닌 **acknowledgment**에 명시한다.

## 2. 학술지 선정 — Predatory Journal 회피

### 2-1. 검증 도구 사용 순서 (권장)

| 순서 | 도구 | 역할 | URL |
|------|------|------|-----|
| 1 | **Think. Check. Submit.** | 7가지 체크리스트로 신뢰성 자가 평가 | https://thinkchecksubmit.org/journals/ |
| 2 | **DOAJ** (open access만) | 등재 여부 확인 — 등재되었다면 일정 수준 검증 통과 | https://doaj.org/ |
| 3 | **KCI** (국내) | 등재학술지 검색 (등재 / 등재후보 / 미등재) | https://www.kci.go.kr/ |
| 4 | **JCR / SCImago** | IF / SJR / Q1-Q4 등급 확인 | https://www.scimagojr.com/ |
| 5 | **Cabells Predatory Reports** (유료) | 60+ 행동 지표로 위험 학술지 식별 | https://www2.cabells.com/ |

> **주의:** 본 스킬은 특정 학술지의 predatory 여부를 단정하지 않는다. Beall's list는 2017년 운영 중단되었고 후속본도 비판받는다. **반드시 위 도구로 직접 검증**한다.

### 2-2. Think. Check. Submit. 7대 체크리스트 (요약)

1. **신뢰성** — 학술지/저자/출판사를 쉽게 확인할 수 있는가
2. **출판사 식별** — 연락처(전화·이메일·우편)가 명확한가
3. **동료심사 투명성** — 독립 심사자, 심사자 수, "수락 보장" 광고 없음
4. **색인/보관** — 검색 가능 데이터베이스(DOAJ, Scopus, KCI 등)에 색인되는가
5. **수수료 명시** — APC, 감면 정책이 사전 공개되는가
6. **저자 지침** — 라이선스/저작권/이해상충 정책 명시
7. **산업 단체 가입** — COPE, DOAJ, OASPA 회원 여부

**대부분 "예"로 답할 수 있을 때만 투고**한다.

## 3. 투고서(Cover Letter)·초록·키워드·제목

### 3-1. Cover Letter 표준 구조 (영문 학술지)

Springer Nature 권장 구조 (1페이지 이내):

```
[헤더] 날짜, 편집장 이름·학술지명

[1단락] 논문 제목, 논문 유형(research article / review / case study), 연구 배경·연구 질문

[2단락] 연구 수행 내용 요약, 주요 발견, 학술적 중요성

[3단락] 학술지 scope 적합성 — "이 연구가 왜 본 학술지 독자에게 흥미로운가"

[4단락] 윤리 선언:
  - 미게재·미투고 확인
  - 모든 저자 동의
  - 이해상충 없음(또는 명시)
  - (해당 시) 학위논문 일부 사용 사실 공시 (§4)

[마무리] 교신저자 연락처, 학술지 특정 요구사항 준수 명시
```

**Elsevier 추가 권장:**
- 자금 정보, 추천/배제 심사위원은 **Cover letter에 넣지 않는다** — 시스템에서 별도 입력칸으로 받는다.
- 1페이지 이내, 핵심 기여 위주.

### 3-2. Cover Letter 영문 템플릿 예시

```
Dear Dr. [Editor's Surname],

I am pleased to submit our original research article titled
"[Manuscript Title]" for consideration as a [Article Type] in
the Journal of Moral Education.

In this study, we [1-2 sentences on what was done]. Our main
findings indicate that [1-2 sentences on key results].
This work contributes to [field] by [significance].

We believe this manuscript fits the scope of Journal of Moral
Education because [link to journal aims — character education,
moral development, etc.]. Readers interested in [topic] will
find [specific contribution] particularly relevant.

We confirm that this manuscript has not been published elsewhere
and is not under consideration by another journal. All authors
have approved the manuscript and agree with its submission to
Journal of Moral Education. The authors declare no conflicts of
interest. [If applicable: Portions of this work appeared in the
first author's master's/doctoral thesis at [Institution, Year];
this has been disclosed in the Acknowledgments section.]

Thank you for considering our submission.

Sincerely,
[Your name]
[Affiliation]
[ORCID iD]
[Email]
```

### 3-3. 한국 KCI 등재지 투고서

- 대부분 학회가 JAMS 또는 자체 시스템에 **양식**을 제공한다 (별도 양식 없을 수도 있음).
- 양식이 있으면 그대로 따른다. 없으면 위 영문 구조를 한국어로 축약 번역해 사용.
- 학위논문 일부 사용 시 반드시 **학술지 발행 학회의 투고 규정** 확인 — 일부 학회는 명시적 동의 필요.

### 3-4. 제목·초록·키워드 최적화

| 요소 | 권장 |
|------|------|
| **제목** | 검색 키워드 포함, 동사·약어 최소화. 학술지마다 글자 수 제한 다름(보통 50–100자). 연구 대상·주제·접근법이 드러나야 함. |
| **초록** | 학술지 양식 확인 — 구조화(Background/Methods/Results/Conclusions) vs 비구조화. KCI 등재지는 비구조화 150–250단어가 일반적이지만 **반드시 해당 학술지 규정 확인**. |
| **키워드** | 학술지 지정 갯수 (보통 3–7개). 분야 표준 용어(thesaurus, ERIC descriptors 등) 사용. 제목과 중복 키워드는 피해 검색 노출 면적을 늘림. |

## 4. 투고 윤리 — 사전 점검 사항

### 4-1. 동시 투고 금지 (One Journal at a Time)

ICMJE / COPE 공통 원칙. 결정 통보(또는 자진 철회 확인) 전에는 다른 학술지에 투고할 수 없다.

### 4-2. 자기표절·학위논문 사용

- 학위논문에 기반한 논문 출판은 **대부분 학술지에서 허용**되는 일반적 관행이다 (Editage / COPE 입장).
- **단, 반드시 cover letter에 명시**하고 본문에서 학위논문을 인용한다.
- 기관 리포지토리에 학위논문이 공개되어 있다면 더더욱 사전 공시 필수.
- 텍스트 재사용은 COPE 기준 "**undisclosed text recycling은 비윤리적**" — 출처를 밝히면 적정 수준에서 허용.
- 학술지마다 정책이 다르므로 **해당 학술지 "duplicate publication" 정책 확인** 필수.

### 4-3. AI 활용 명시 (COPE 2024 입장)

| 사용 유형 | 표시 위치 | 표시 내용 |
|----------|----------|-----------|
| 새 텍스트 초안 작성에 AI 사용 | Acknowledgments | 사용 도구·버전, 사용한 prompt |
| 분석/표/그림/코드 생성 | Methods + Abstract | 도구·버전·prompt·시간·날짜 |
| 단순 문법·맞춤법 교정 (Grammarly 등) | (학술지 정책에 따라) 다수 학술지가 면책 | 정책 확인 후 결정 |

> **AI는 저자가 될 수 없다** (COPE 입장). 모든 AI 생성 내용에 대한 책임은 저자가 진다.

## 5. 심사 결과 유형 이해

### 5-1. 영문 학술지 일반

| 결과 | 의미 | 대응 |
|------|------|------|
| **Accept** (조건 없는 게재) | 매우 드물다 | 게재 절차 진행 |
| **Accept with minor revision** | 첫 결과로 가장 바람직 | §6 response letter (간결) |
| **Major revision** | 본격 수정 필요, 재심 가능성 |  §6 response letter (상세) |
| **Revise and resubmit** | 사실상 새 투고 — 새 manuscript ID 부여 가능 | 대폭 재작성 후 재투고 |
| **Reject** | 부적합 또는 결정적 결함 | §8 — 다른 학술지 재투고가 일반적, appeal은 신중 |
| **Desk reject** | 편집장 단계에서 scope 불일치·형식 미준수로 반려 | scope·형식 재점검 후 다른 학술지 |

### 5-2. 한국 KCI 등재지 (4단계 일반)

| 결과 | 의미 |
|------|------|
| **게재가** | 그대로 게재 (드물다) |
| **수정 후 게재** | 경미한 수정 후 게재 — 보통 재심 없이 편집위 확인 |
| **수정 후 재심** | 수정 후 재심사 필수 — 동일 또는 신규 심사위원이 재검토 |
| **게재불가** | 게재 불가, 사유 명시됨 |

> 학회별로 5단계(예: 무수정게재/수정후게재/수정후재심/대폭수정후재심/게재불가)로 세분되기도 한다. 학술지 규정 확인.

### 5-3. 결과 도착 시 행동

1. **24–48시간 묵혀두기** — 감정적 반응 방지
2. 모든 코멘트를 **번호 매겨 표 형태로 정리** (Excel/Markdown)
3. **수정 가능 여부 분류**: 동의 / 부분 동의 / 이견
4. 마감일 확인 (보통 minor 4주, major 8–12주)
5. 마감 못 맞추면 **사전에** 편집장에게 연장 요청

## 6. Revision Response Letter 작성 — 핵심 스킬

### 6-1. 표준 구조

```
[헤더]
Dear Dr. [Editor's Surname],

We thank the editor and the two reviewers for their thoughtful
and constructive comments on our manuscript "[title]"
(Manuscript ID: XXX). We have revised the manuscript
addressing all the points raised. Below we provide a
point-by-point response. The corresponding changes in the
manuscript are highlighted in [yellow / track changes / blue text].

[Summary of major changes — 5-7줄로 핵심 변경 요약]

[Reviewer 1]

Comment 1.1: "심사위원 코멘트 원문 그대로 인용 (이탤릭 또는 indent)"

Response: We thank the reviewer for this important point.
We agree that [...]. Accordingly, we have revised
[section/paragraph] on page X, lines Y–Z, to read:

> "[수정된 본문 발췌]"

Comment 1.2: "..."

Response: ...

[Reviewer 2]

Comment 2.1: "..."

Response: ...

[Editor]

Editor Comment 1: "..."
Response: ...

We hope these revisions adequately address the concerns raised
and that the manuscript is now suitable for publication in
[Journal Name].

Sincerely,
[Authors]
```

### 6-2. 작성 5대 원칙

1. **모든 코멘트에 응답한다** — 빠뜨린 코멘트는 reject 사유.
2. **코멘트를 원문 그대로 인용** — paraphrase 금지.
3. **응답은 편집장에게** — "We thank the reviewer", "We agree with the reviewer" (3인칭). "you"는 편집장에게만.
4. **페이지·줄 번호 명시** — "page 7, lines 142–145" 형식.
5. **수정된 본문 발췌 포함** — 편집장이 PDF 열지 않고도 변경 확인 가능하도록.

### 6-3. 응답 패턴 3가지

| 상황 | 응답 패턴 |
|------|----------|
| **동의 + 수정** | "We thank the reviewer for this insightful comment. We agree that ... We have revised ... (page X, lines Y)" |
| **부분 동의** | "We appreciate this suggestion. We agree on [부분]. However, regarding [부분], we believe ... [근거]. We have therefore revised X but kept Y." |
| **이견 (§7 참고)** | "We thank the reviewer for raising this point. We respectfully disagree because ... [학술적 근거]. We have added a clarifying sentence on page X to prevent misunderstanding." |

### 6-4. Track Changes vs Highlight

| 학술지 요구 | 제출 파일 |
|------------|----------|
| Track changes 요구 | Word `변경 내용 추적` 켠 채 저장한 파일 |
| Highlight 요구 | 수정 부분 노란색 형광펜 표시 (또는 색상 텍스트) |
| Clean version 요구 | 수정 표시 없는 최종본 — 보통 track changes 버전과 함께 둘 다 제출 |

> 학술지 지침에 명시되지 않으면 **두 버전(track + clean) 모두 제출**이 안전하다.

## 7. 이견(Disagreement) 표명 방법

### 7-1. 언제 이견을 표명하는가

| 상황 | 표명 가능 |
|------|:---:|
| 심사위원이 사실관계를 잘못 이해 | ✅ 정정 필요 |
| 권장 변경이 핵심 결과를 훼손 | ✅ 학술적 근거로 거절 |
| 추가 실험·자료가 윤리·자원·시간 제약상 불가 | ✅ 제약 설명 |
| 단순히 동의하고 싶지 않음 | ❌ 동의해서 수정 권장 |

### 7-2. 표명 표현 예시 (영문)

```
"We respectfully disagree with the reviewer on this point because [근거].
Specifically, [선행연구 / 데이터 / 방법론적 정당화 인용]. To prevent
the misunderstanding, we have clarified this point on page X, lines Y–Z."
```

```
"We appreciate the reviewer's suggestion. However, conducting [추가 분석]
would require [데이터/시간/윤리 승인], which is beyond the scope of the
present study. We have noted this as a limitation on page X and as a
direction for future research on page Y."
```

### 7-3. 한국어 표현 예시 (KCI 등재지)

```
"심사위원님의 지적에 감사드립니다. 다만 본 연구에서는 [근거: 선행연구
A(Author, Year), 본 연구의 자료 한계 등]에 따라 [원래 입장]을 유지하는
것이 적절하다고 판단됩니다. 다만 심사위원님의 우려를 반영하여 X페이지
Y행에 다음과 같이 부연 설명을 추가하였습니다: '...'."
```

### 7-4. 이견 표명의 절대 금지 사항

- 비꼬기·수사적 질문·감정적 표현 금지
- 심사위원의 자격·전문성 비판 금지
- "obvious", "clearly wrong" 같은 단정적 단어 금지
- 답변하지 않고 넘어가기 금지 (편집장 즉각 인지)
