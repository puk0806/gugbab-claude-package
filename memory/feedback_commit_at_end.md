다단계(Tier-based) 작업 세션에서는 **중간에 커밋하지 않는다.** 모든 단계를 끝낸 뒤 사용자가 직접 커밋 명령을 내릴 때(또는 사용자가 명시 승인한 마지막 검증 직후) 한 번에 커밋한다.

**Why:** 2026-04-24 `@gugbab/react` 30 컴포넌트 + Radix-parity 고도화 세션에서 사용자가 두 차례 커밋 시도를 인터럽트하며 "전부 다 처리한 거야?" 형태로 스코프 확인 → "전부 다 일단 만들어줘" / "커밋은 전부다 처리한거야 헤드리스 컴포넌트들??? / 마지막에 할거야" 신호. 단계별 커밋이 검증되지 않은 산출물을 영구 기록으로 남기는 걸 사용자는 원하지 않음.

**How to apply:**

- Tier·Phase 단위로 진행하는 큰 작업 → 각 Tier 끝마다 **테스트만 검증**하고 커밋은 하지 않음
- 모든 Tier 완료 + 최종 verification(test/typecheck/build/biome) 통과 후 사용자 명시 명령("커밋 푸쉬해줘" 등) 받을 때 커밋
- 중간에 사용자가 "다 끝난 거야?" / "전부 처리한 거야?"라고 물으면 **솔직한 진척률 + 격차 리스트**로 답하고 커밋 시도하지 않음
- 단발성 변경(에이전트·스킬 1개, 버그 픽스 등)은 기존대로 커밋 분리 원칙(`.claude/rules/git.md`) 적용 — 본 규칙은 **누적형 다단계 세션에만 적용**

**참고:** `feedback_batched_execution.md`의 "초반 방향만 대화 후 일괄 진행"과 짝을 이룸. 그쪽이 설계 단계 단축이라면, 본 메모리는 구현·커밋 단계 단축.
