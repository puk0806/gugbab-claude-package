`@gugbab/tokens` 같은 디자인 시스템 패키지는 **외부 디자인 라이브러리(MUI, Radix, Tailwind, Chakra 등)를 빌드 타임에서도 의존하지 않는다.** 토큰 값은 정적 객체(`ThemeTokens`)로 코드에 박제하고, 외부 라이브러리는 일회성으로 값 베껴오는 reference로만 활용한다.

**Why:** 2026-04-29 사용자가 명시적으로 "MUI, Radix 쪽에 변경이 이루어지면 나도 변하는거잖아 — 의존성을 없애고 토큰 자체를 내가 보관"이라고 표명. 이전엔 `createTheme()` / `@radix-ui/colors`를 빌드 타임 어댑터로 호출해 토큰 추출 → 외부 변경이 우리 디자인에 자동 전파됨. 사용자는 의도적 갱신만 허용하고 외부 변동에 휘둘리는 것을 거부.

**How to apply:**

- 새 디자인 토큰 / 색·타입 시스템 추가 시 외부 라이브러리를 dep으로 두지 말 것. 정적 객체로 박제.
- "어댑터" 패턴(외부 라이브러리에서 동적 추출)이 필요해 보이는 경우, 일회성 dump 스크립트로 값만 추출 후 정적 모듈에 박제. 외부 dep는 추출 시점만 잠시 사용.
- 외부 디자인 라이브러리가 업데이트(MUI 7, Radix Themes 4 등)했을 때 적용 여부는 **수동 결정**. 자동 추적 X.
- 정적 토큰 갱신은 별도 PR로 진행하여 디자인 변경 의도를 git history에 명시.
- 다른 패키지에서도 동일 — 외부 라이브러리에 우리 코드가 끌려가지 않게 가능한 한 정적·자기완결.

**참고:** `feedback_batched_execution`, `feedback_commit_at_end`와 같은 톤. 사용자는 통제권을 본인이 갖기를 원하고, 자동 변경/외부 영향을 최소화한다.
