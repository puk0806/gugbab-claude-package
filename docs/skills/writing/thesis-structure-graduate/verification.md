## 4. 검증 체크리스트 (Test List)

### 4-1. 내용 정확성
- [✅] 공식 문서·공인 자료와 불일치하는 내용 없음
- [✅] 적용 범위(한국 일반대학원, 인문·교육 계열)가 명시되어 있음
- [✅] deprecated된 절차·제도를 권장하지 않음
- [✅] 절차·체크리스트가 실제 활용 가능한 형태임

### 4-2. 구조 완전성
- [✅] YAML frontmatter 포함 (skill, category, version, date, status)
- [✅] 소스 URL과 검증일 명시
- [✅] 핵심 개념 설명 포함 (석사·박사 차이, 표준 구조, 절차)
- [✅] 단계별·항목별 체크리스트 포함
- [✅] 언제 사용 / 언제 다른 스킬을 참조할지 기준 포함 (research-ethics-and-integrity 연계)
- [✅] 흔한 실수 패턴 포함 (도덕윤리교육 특수 사례 5종)

### 4-3. 실용성
- [✅] 대학원생이 참조했을 때 연구 단계별로 실제 도움이 되는 수준
- [✅] 지나치게 이론적이지 않고 한국 대학원 실제 관행 반영
- [✅] 범용적 (도덕윤리교육 외 인문·교육 계열에도 적용 가능)
