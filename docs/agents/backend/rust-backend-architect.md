# rust-backend-architect

## 개요

- **역할**: Rust + Axum 백엔드 아키텍처 설계 전담 에이전트. 크레이트 구조, 레이어드 아키텍처, 트레이트 추상화, 에러 계층, 성능/보안 트레이드오프 등 아키텍처 수준의 판단과 결정을 담당한다.
- **모델**: opus
- **도구**: Read, WebSearch, WebFetch
- **카테고리**: backend

## 사용 시점

- Rust + Axum 프로젝트의 크레이트/워크스페이스 구조를 설계할 때
- 레이어드 아키텍처(handler/service/repository) 분리 방법을 결정할 때
- 에러 타입 계층 설계 (thiserror vs anyhow 선택 등)
- 트레이트 기반 추상화, 의존성 주입 패턴 결정
- 미들웨어/추출자 배치 전략 수립
- 기술 스택 선택 시 근거 있는 비교가 필요할 때
- 성능/보안 트레이드오프 판단이 필요할 때

## 사용 예시

- "Axum 프로젝트에서 워크스페이스와 크레이트를 어떻게 분리해야 할까?"
- "에러 타입 계층을 어떻게 설계하면 좋을까? thiserror vs anyhow 선택 기준은?"
- "인증/인가 미들웨어를 어디 레이어에 배치해야 하지?"

## 입력/출력

- **입력**: 아키텍처 관련 질문, 프로젝트 규모/제약 조건, 현재 코드베이스 구조 (있는 경우)
- **출력**: 판단 근거가 명시된 아키텍처 제안, 구체적 구조 예시(폴더/모듈 트리, Cargo.toml 의존 관계), 트레이드오프 비교표, 단계별 적용 순서

## 관련 에이전트

- **rust-backend-developer** (backend) -- 설계된 아키텍처를 실제 코드로 구현하는 에이전트
- **build-error-resolver** (backend) -- 아키텍처 변경 후 발생하는 빌드 에러 해결
- **business-domain-analyst** (domain) -- DDD 기반 도메인 모델이 아키텍처에 반영될 때 협업
